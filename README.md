# _Полностью автоматизированный парсер Яндекс Карт_
Данный проект представляет с собой _консольную программу_ по парсингу Yandex Maps в двух версиях, ручная и автоматическая.
### Кратко чем две версии отличаются друг от друга и новые изменения:
#### Чем отличаются:
Ручная версия предполагает, что вы прокрутите карточки в яндекс картах сами, потом сохраните страницу в формат
`*.htm` в директорию и запустите. Автоматическая версия будет прокручивать карточки сама, запустив хромдрайвер.
Ручная версия более надёжнее в плане работоспособности и поиска максимального числа карточек, однако неудобна.
В свою очередь, автоматическая версия избавит вас от неудобства, но взамен возможны ошибки.
#### Что нового:
>1) Убран файл `*.exe` и добавлены _bat-ники_ `create_venv.bat` создаёт виртуальную среду и устанавливает зависимости (запускать нужно один раз),
а `parser.bat` и `parser_for_hand.bat` запускает скрипты. 
>2) Обновлён код, который соответствует новой версии библиотеки [Selenium](https://www.selenium.dev/documentation/webdriver/getting_started/first_script/?language=python),
и создаётся отдельный файлик `*.txt` для ссылок на сайты организаций (вроде только в ручной версии). 
>3) Обновлены `class` в `div` для поиска по новой `html-странице` яндекс карт. 
>4) Добавлены опции "Парсить ли фото и логотипы организаций или нет" и "Коэфицент (время между прокруткой страницы) точности" (это только для автоматической версии)

#### Состоит из трёх файлов:
>1) `Main.py` - является главным запускаемым файлом. Именно его запускет `*.bat`.
>2) `Write_Data.py` - Файл по обработке данных с `html-старницы`, через библиотеки `Python` (`BeautifulSoup`).

### _Что используется для этого парсера_
Язык программирования `Python`, версии 3.12. 

**Библиотеки:**
>* BeautifulSoup ( Для разбора и выдёргивания информации из `HTML-разметки` )

# _Как работает данный парсер_
Скрипт на вход принимает данные. Программа спрашивает, о работе в фоновом режиме. Дальше
спрашивает какой город будет парситься, и что именно парсить.

Когда скрипт выполняется, код с помощью `Selenium` вводит в поиск город и что надо найти.
После этого запускается цикл `while`, и код каждый раз скролит страницу вниз, сравнивая массив ссылок до прокрутки и после.
Когда число ссылок до и после совпадает, парсер ссылок на карточки останавливается. **_Это всё работает только для автоматической версии_**.

Когда первая часть выполнена: сбор всех ссылок на все карточки, то начинается вторая часть.
`Selenium` заходит на все ссылки, берёт у них `html` страницы, и через `BeautifulSoup` достаёт необходимые данные, 
даже собирает все фотки и логотипы в отдельные папки. А после создаёт `csv-файл` и записывает данные туда.

На выходе имеется один `csv-файл` и папки с фотками логотипов и картинок с карточки ( у папки с картинками название  - id самой карточки ).
Программа не идеальна, много вылетает, может выдавать, ошибки не из чего, но всё же большиство раз она запускалась благополучно. 
Сейчас добавлено ещё больше конструкций `try-except`, и программа работает намного стабильнее.
